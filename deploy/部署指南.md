
# AI 聊天平台 — 生产环境部署指南

> 适用系统：Linux (Ubuntu 20.04+ / CentOS 7+)  
> 所需软件：Java 21+、MySQL 8.0+、Nginx 1.18+

---

## 一、部署架构

```
浏览器 (HTTPS)
    │
    ▼
┌─────────────────────────────────────┐
│  Nginx (:80 / :443)                 │
│                                      │
│  /api/*  → proxy_pass → Backend     │
│  /*      → try_files   → Frontend   │
│                                      │
│  ★ 统一入口，消除跨域                 │
└──────┬──────────────────────────────┘
       │                    │
       ▼                    ▼
┌──────────────┐   ┌──────────────────┐
│ SpringBoot   │   │ 前端静态文件       │
│ (:8080)      │   │ (dist/ 目录)      │
│              │   │                  │
│ /api/user/*  │   │ /index.html      │
│ /api/chat/*  │   │ /assets/*.js     │
│ /api/session/*│  │ /assets/*.css    │
│ /api/health  │   │ /favicon.svg     │
└──────┬───────┘   └──────────────────┘
       │
       ▼
┌──────────────┐
│ MySQL (:3306)│
└──────────────┘
```

**关键点：** 前后端同域名同端口，API 请求走相对路径 `/api/`，不再需要 CORS。

## 三、部署步骤

### 步骤 1：环境准备

```bash
# 1.1 更新系统
sudo apt update && sudo apt upgrade -y    # Ubuntu
# sudo yum update -y                      # CentOS

# 1.2 安装 Java 21
java -version  # 确认版本 ≥ 21

# 1.3 安装 Nginx
sudo apt install nginx -y                 # Ubuntu
# sudo yum install nginx -y               # CentOS
nginx -v  # 确认版本 ≥ 1.18

# 1.4 安装 MySQL 8.0（如未安装）
sudo apt install mysql-server -y
```

### 步骤 2：配置 MySQL

```bash
# 2.1 创建数据库和表
mysql -u root -p < /opt/ai-chat/backend/sql/init.sql

# 2.2 创建应用专用用户（生产环境不要用 root）
mysql -u root -p
```

```sql
CREATE USER 'ai_chat_app'@'localhost' IDENTIFIED BY '生成一个强密码';
GRANT SELECT, INSERT, UPDATE, DELETE ON ai_chat.* TO 'ai_chat_app'@'localhost';
FLUSH PRIVILEGES;
```

### 步骤 3：设置环境变量

```bash
# 编辑 /etc/profile 或 /etc/environment

# ---- 数据库 ----
export MYSQL_HOST=localhost
export MYSQL_PORT=3306
export MYSQL_DATABASE=ai_chat
export MYSQL_USERNAME=ai_chat_app
export MYSQL_PASSWORD=你的数据库密码

# ---- JWT ----
# 生成方式：openssl rand -base64 48
export JWT_SECRET=生成的64位随机字符串

# ---- DeepSeek ----
export DEEPSEEK_API_KEY=sk-你的API密钥

# 使环境变量生效
source /etc/profile
```

### 步骤 4：部署后端

```bash
# 4.1 上传 JAR 包
scp target/ai-chat-1.0.0.jar user@server:/opt/ai-chat/backend/

# 4.2 创建启动脚本
cat > /opt/ai-chat/backend/start.sh << 'SCRIPT'
#!/bin/bash
cd /opt/ai-chat/backend
nohup java -jar ai-chat-1.0.0.jar \
    --spring.profiles.active=prod \
    > app-startup.log 2>&1 &
echo "PID: $!"
SCRIPT
chmod +x /opt/ai-chat/backend/start.sh

# 4.3 启动
/opt/ai-chat/backend/start.sh

# 4.4 验证
curl http://localhost:8080/api/health
# 预期返回: {"code":200,...,"data":{"status":"UP","database":"UP"}}
```

### 步骤 5：部署前端

```bash
# 5.1 本地构建（在开发机上）
cd frontend
npm run build

# 5.2 上传到服务器
scp -r dist/* user@server:/opt/ai-chat/frontend/
```

### 步骤 6：配置 Nginx

```bash
# 6.1 复制配置文件
sudo cp /opt/ai-chat/deploy/nginx.conf /etc/nginx/sites-available/ai-chat

# 6.2 修改域名（如果有的话）
sudo nano /etc/nginx/sites-available/ai-chat
# 修改 server_name 为你的域名或 IP

# 6.3 创建软链接启用站点
sudo ln -s /etc/nginx/sites-available/ai-chat /etc/nginx/sites-enabled/

# 6.4 测试配置语法
sudo nginx -t
# 应输出: syntax is ok / test is successful

# 6.5 重载 Nginx
sudo systemctl reload nginx
```

### 步骤 7：验证部署

```bash
# 7.1 检查前端
curl http://localhost/
# 应返回 index.html 内容

# 7.2 检查 API（通过 Nginx）
curl http://localhost/api/health
# 应返回健康检查 JSON

# 7.3 检查 SPA 路由（直接访问前端子路由应返回 index.html）
curl http://localhost/login
# 应返回 index.html 内容（非 404）

# 7.4 检查注册（端到端）
curl -X POST http://localhost/api/user/register \
  -H "Content-Type: application/json" \
  -d '{"username":"deploytest","password":"Test1234","nickname":"test"}'
```

## 四、配置认证

### 关键设计：生产环境不再需要 CORS

| | 开发环境 | 生产环境 |
|------|---------|---------|
| 前端地址 | `http://localhost:5173` | `https://你的域名` |
| API 地址 | `http://localhost:8080/api` | `/api`（相对路径） |
| 跨域？ | 是（不同端口）→ 需要 CorsConfig | 否（同源）→ Nginx 代理 |
| CorsConfig | 生效 | 保留代码但不起作用 |

**原因：** Nginx 统一代理后，浏览器访问 `https://域名/api/xxx` 和 `https://域名/index.html`，协议+域名+端口完全相同，不触发跨域检查。

### SSE 流式聊天特殊配置

流式聊天使用 `text/event-stream`，需要：
- `proxy_buffering off` — 关闭 Nginx 缓冲
- `proxy_read_timeout 300s` — 延长读取超时
- `proxy_http_version 1.1` — HTTP/1.1 长连接
- `chunked_transfer_encoding on` — 分块传输

这些已在 nginx.conf 中配置好。

## 五、常用运维命令

```bash
# 查看后端进程
ps aux | grep java

# 重启后端
kill [PID] && /opt/ai-chat/backend/start.sh

# 查看后端日志
tail -f /opt/ai-chat/backend/logs/ai-chat.log

# 重载 Nginx（修改配置后）
sudo nginx -t && sudo systemctl reload nginx

# 查看 Nginx 日志
tail -f /var/log/nginx/ai-chat-access.log
tail -f /var/log/nginx/ai-chat-error.log

# 数据库备份
mysqldump -u root -p ai_chat > /backup/ai_chat_$(date +%Y%m%d).sql
```

# AI 聊天平台 — 项目总结与学习笔记

> 项目基于 SpringBoot 3.2 + Vue 3 + DeepSeek API 构建的 AI 聊天平台

---

## 1. 今天项目做了什么

从零开始搭建了一个完整的 AI 聊天平台，包含以下 **6 个步骤**：

| 步骤 | 内容 | 状态 |
|------|------|------|
| **Step 1** | 创建 SpringBoot 后端项目结构（Maven + 目录分层） | ✅ |
| **Step 2** | 创建 MySQL 数据库和用户表 | ✅ |
| **Step 3** | 实现用户注册功能（用户名唯一性校验 + BCrypt 加密） | ✅ |
| **Step 4** | 实现用户登录 + JWT 令牌认证 | ✅ |
| **Step 5** | 实现 AI 聊天接口（对接 DeepSeek API） | ✅ |
| **Step 6** | 搭建 Vue3 前端项目（登录/注册/聊天页面） | ✅ |

**遇到的问题及解决：**
1. **Lombok 与 Java 25 不兼容** → 手动编写 getter/setter
2. **BCrypt 类找不到** → 添加 `spring-security-crypto` 依赖
3. **CORS 跨域问题** → 创建 `CorsConfig.java` 允许前端域名
4. **DeepSeek API 连接超时** → 配置 RestTemplate 超时时间

### 后端核心文件

#### `pom.xml` — 项目依赖管理

### 前端核心文件

#### `api/index.js` — Axios 封装
```javascript
// 请求拦截器：自动添加 JWT 令牌
axios.interceptors.request.use(config => {
    const token = localStorage.getItem('token');
    if (token) config.headers.Authorization = `Bearer ${token}`;
    return config;
});
// 响应拦截器：统一错误处理
axios.interceptors.response.use(res => {
    if (res.data.code === 401) {  // 未登录
        router.push('/login');
    }
});
```

#### `router/index.js` — 路由守卫
```javascript
router.beforeEach((to, from, next) => {
    const token = localStorage.getItem('token');
    if (to.path === '/chat' && !token) {
        next('/login');  // 未登录跳转登录页
    } else {
        next();
    }
});
```

## 5. 你应该重点理解什么

### 🔥 最高优先级（必须掌握）

1. **三层架构（Controller → Service → Mapper）**
   - 这是 JavaWeb 开发的核心架构模式
   - Controller：接收请求、返回响应（不写业务逻辑）
   - Service：编写业务逻辑（注册校验、密码加密）
   - Mapper：数据库操作（MyBatis Plus 自动实现）

2. **JWT 认证流程**
   ```mermaid
   graph LR
       A[用户登录] --> B[后端验证密码]
       B --> C[生成 JWT Token]
       C --> D[返回给前端]
       D --> E[前端存 localStorage]
       E --> F[请求携带 Token]
       F --> G[后端验证 Token]
       G --> H[返回数据]
   ```

3. **前后端如何通信**
   - 前端 Axios 发送 HTTP 请求到后端 API
   - 后端返回 JSON 格式的统一响应（Result）
   - 跨域问题通过 CorsConfig 解决

### 📌 中等优先级（需要理解）

4. **MyBatis Plus 的使用**
   - `BaseMapper<T>` 提供 `insert/select/update/delete` 方法
   - `LambdaQueryWrapper` 构建查询条件（eq、orderBy）
   - `@TableName`、`@TableId`、`@TableField` 注解

5. **BCrypt 密码加密**
   - 密码不能明文存储，必须哈希加密
   - `BCrypt.hashpw(明文, BCrypt.gensalt())` — 注册时加密
   - `BCrypt.checkpw(明文, 哈希)` — 登录时验证

6. **RestTemplate 调用第三方 API**
   - 后端作为客户端调用 DeepSeek API
   - 需要配置超时时间（避免无限等待）
   - 需要处理异常（网络问题、API 错误）

### 👀 了解即可

7. **DTO 模式** — 请求/响应对象与数据库实体分离
8. **CORS 配置** — 允许不同端口的前后端通信
9. **Vue Router 导航守卫** — 未登录自动跳转
10. **Axios 拦截器** — 自动携带 Token、统一错误处理

## 核心流程

### 注册流程
```
用户提交表单 → UserController → UserService.register()
  → 检查用户名是否已存在 → BCrypt 加密密码
  → 插入数据库 → 返回成功
```

### 登录流程
```
用户提交表单 → UserController → UserService.login()
  → 查询用户 → BCrypt 验证密码
  → 生成 JWT Token → 返回 Token + 用户信息
```

### 聊天流程
```
用户发送消息 → ChatController → 解析 JWT 获取 userId
  → DeepSeekService.chat()
  → 保存用户消息到 DB
  → 获取最近 10 条聊天记录
  → 调用 DeepSeek API
  → 保存 AI 回复到 DB
  → 返回 AI 回复
```

## 关键代码片段

### 1. JWT 生成
```java
public String generateToken(Long userId, String username) {
    return Jwts.builder()
        .subject(userId.toString())
        .claim("username", username)
        .issuedAt(new Date())
        .expiration(new Date(System.currentTimeMillis() + jwtConfig.getExpiration()))
        .signWith(Keys.hmacShaKeyFor(jwtConfig.getSecret().getBytes()))
        .compact();
}
```

## 常见问题

### Q: 为什么不用 Lombok？
A: Java 25 与 Lombok 不兼容，编译时报错找不到 getter/setter。解决方案是手动编写。

### Q: 为什么需要 CORS 配置？
A: 前端运行在 localhost:5173，后端在 localhost:8080，不同端口属于跨域。浏览器会阻止跨域请求，需要后端配置允许。

### Q: 为什么 RestTemplate 会超时？
A: 默认 RestTemplate 没有超时设置，遇到网络波动会无限等待。需要手动配置 connectTimeout 和 readTimeout。

### Q: JWT 和 Session 有什么区别？
A: Session 存储在服务器内存（有状态），JWT 存储在客户端（无状态）。JWT 适合分布式系统，不需要共享 Session。
